## 🗺️ Recommended Implementation Roadmap

### Phase 1: Core Engagement (Immediate — 1-2 Sprints)
> Focus: Make the existing game more engaging and addictive

1. **[002 — Achievement Bonuses](./002-achievement-bonuses.md)** *(0.5 days)* — Highest ROI change. Transform existing achievements into progression.
2. **[001 — Enhanced Void Rift Types](./001-enhanced-void-rift-types.md)** *(2-3 days)* — Primary active engagement driver. Our "Golden Cookies."
3. **[004 — Generator Synergies](./004-generator-synergies.md)** *(2-3 days)* — Adds strategic depth to generator purchases.
4. **[003 — Active Cooldown Abilities](./003-active-cooldown-abilities.md)** *(3-5 days)* — Fills the "what do I do?" gap.

### Phase 2: Retention & Depth (Short-term — 2-4 Sprints)
> Focus: Keep players coming back and give them more to explore

5. **[007 — Returning Player Bonuses](./007-returning-player-bonuses.md)** *(1-2 days)* — Improve D1/D7 retention.
6. **[012 — Expanded Research Tree](./012-expanded-research-tree.md)** *(3-4 days)* — Double content via branching paths.
7. **[010 — Secrets & Easter Eggs](./010-secrets-and-easter-eggs.md)** *(2-3 days)* — Community building through discovery.
8. **[009 — Daily Login Rewards](./009-daily-login-rewards.md)** *(1-2 days)* — Daily engagement habit.

### Phase 3: Strategic Expansion (Medium-term — 3-6 Sprints)
> Focus: Add systems that transform how the game plays

9. **[005 — Second Prestige Layer](./005-second-prestige-layer.md)** *(5-8 days)* — Triple game lifetime.
10. **[006 — Challenge Run System](./006-challenge-run-system.md)** *(4-6 days)* — Replayability through restrictions.
11. **[013 — Void Forge](./013-void-forge-resource-conversion.md)** *(2-3 days)* — Resource management depth.
12. **[011 — Reactor Cosmetics](./011-reactor-cosmetics.md)** *(3-5 days)* — Visual personalization and aspiration.

### Phase 4: Endgame & Mastery (Long-term — 6+ Sprints)
> Focus: Create infinite replayability and endgame depth

13. **[014 — Void Expeditions](./014-void-expeditions.md)** *(7-10 days)* — Second gameplay loop.
14. **[008 — Seasonal Events](./008-seasonal-events-system.md)** *(5-7 days/event)* — Recurring engagement.
15. **[015 — Ascendancy Paths](./015-ascendancy-paths.md)** *(10-15 days)* — Three games in one.
16. **[016 — Endgame: Void Singularity](./016-endgame-void-singularity.md)** *(10-15 days)* — Infinite scaling, true ending, competitive benchmarks.

## 🔑 Key Design Principles

These proposals follow consistent design principles drawn from the [IDLE_GAME_DESIGN_RESEARCH](../../IDLE_GAME_DESIGN_RESEARCH.md):

1. **Config-driven** — All features use config files for balance constants, enabling rapid tuning.
2. **Additive, not replacing** — New features enhance existing systems rather than replacing them.
3. **Progressive disclosure** — Features unlock gradually, preventing overwhelm.
4. **Multiple engagement timescales** — Seconds (clicks/rifts), minutes (abilities/recipes), hours (research/expeditions), days (daily rewards), weeks (seasonal events).
5. **Meaningful choices** — Branching research, ascendancy paths, and challenge strategies create genuine decisions.
6. **Respect player time** — Automators in late game reduce tedium; offline bonuses reward breaks.
7. **Visual spectacle** — Every feature has visual integration with the reactor canvas.
8. **Community-building** — Secrets, challenges, and benchmarks create shareable moments.
